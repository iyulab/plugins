# Issue Response Templates

Templates for each iyu decision type, ready to customize.

## ACCEPT

```markdown
Thank you for this suggestion! [Demonstrate understanding]

This aligns well with [project goal/philosophy], and we're happy to move forward.

**Implementation plan:**
- [Timeline or milestone]
- [Any scope adjustments]

[If applicable: PRs welcome!]
```

**Quick acceptance**: "Great idea - this is exactly what we want. Implemented in [version/PR]."

**With scope adjustment**: "We love the core idea. We'll implement [adjusted version] because [reason]."

## ADAPT

```markdown
Thank you for raising this! [Show understanding of underlying need]

We'd like to address this differently than proposed:

**Our approach:** [Adapted solution]

**Why this works better:**
- [Benefit 1]
- [Benefit 2]
- [How it still solves their problem]

Would this work for you?
```

## DEFER

```markdown
Thank you for this thoughtful suggestion!

This is valuable, but we're currently focused on [priority].

**Why not now:**
- [Specific reason]

**Roadmap placement:** [Where this fits]

**What would accelerate:**
- Community interest/votes
- Sponsored development
- Community PR

Added `[label]` label to track.
```

## REDIRECT

```markdown
Thank you for reaching out! [Show understanding]

This falls outside our scope - we focus on [our focus], not [what they're asking].

**Why we maintain this boundary:**
- [Philosophy reason]
- [Practical reason]

**Alternatives:**
1. **[Tool 1]**: [How it helps]
2. **[Workaround]**: [How to achieve similar result]
3. **[Extension point]**: [How to build on top of us]

Would any of these work?
```

## DECLINE

```markdown
Thank you for taking the time to suggest this!

After consideration, this doesn't align with [project]'s direction.

**Our concern:**
- [Primary reason - specific and respectful]
- [How this conflicts with philosophy]

**What we would welcome instead:**
- [Alternative contribution]
- [Related area]

We appreciate your interest in [project].
```

**Philosophical misalignment variant:**
```markdown
This conflicts with our philosophy of [principle]. We believe [explanation].

**The trade-off:**
- Your suggestion optimizes for [X]
- We optimize for [Y]

Happy to discuss our reasoning further.
```
